# API Specification: Traffic Analysis & Classification System

Данная спецификация описывает интерфейс взаимодействия между модулем захвата трафика (Sniffer), нейросетью для классификации и фронтенд-интерфейсом (Dashboard).

## 1. Основные функции API

| Метод (Функция) | Входные параметры (Input) | Выходные данные (Output) | Описание задачи |
| :--- | :--- | :--- | :--- |
| **`register_new_flow`** | `src_ip`, `src_port`, `dst_ip`, `dst_port`, `protocol`, `size_bytes` | `flow_id` (int) | Регистрация нового «сырого» потока сразу после захвата пакета. Создает запись в логе. |
| **`update_classification`** | `flow_id`, `app_name`, `category_name`, `confidence` | `status` (bool) | **Для нейросети:** обновляет запись результатами анализа (название приложения, категория и уверенность модели). |
| **`get_total_stats`** | `time_range` ('1h', '24h', 'all') | `total_count` (int) | Возвращает общее количество потоков для виджета **Total Flows**. |
| **`get_throughput_data`** | `interval_sec` (int) | `list[{time, mbit_s}]` | Рассчитывает пропускную способность для центрального графика **Traffic Activity**. |
| **`get_flow_logs`** | `limit`, `offset`, `filters` | `list[FlowObject]` | Отдает полные данные для таблицы **Request Log** (нижняя часть интерфейса). |

---

## 2. Жизненный цикл данных (Execution Pipeline)

Система работает по принципу конвейера, где данные проходят три ключевых этапа:

### Этап I: Захват (Trigger: Сниффер)
Как только сетевой интерфейс фиксирует новый поток, вызывается функция `register_new_flow`.
* **Результат:** В базе данных создается запись. В интерфейсе (Figma Dashboard) пользователь мгновенно видит новую строку в таблице со статусом **"Analyzing..."**.

### Этап II: Анализ (Trigger: Нейросеть)
Модуль классификации (Нейросеть) забирает характеристики потока и определяет его принадлежность к конкретному приложению (например, YouTube или Telegram). По завершении вызывается `update_classification`.
* **Результат:** Пустые поля «Приложение» и «Тип» в таблице логов обновляются реальными данными.

### Этап III: Визуализация (Trigger: Запрос Dashboard)
Фронтенд-приложение с периодичностью в 1-5 секунд опрашивает API методами `get_total_stats` и `get_throughput_data`.
* **Результат:** Пользователь видит актуальную нагрузку на сеть в $Mbit/s$ и общее количество обработанных соединений.

---

## 3. Расчетные формулы

Для отображения данных на графике используется формула расчета пропускной способности:

$$Throughput (Mbit/s) = \frac{\sum size\_bytes \cdot 8}{1,048,576 \cdot interval\_sec}$$
